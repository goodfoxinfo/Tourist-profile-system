# Создание туристического профиля

Твоя задача — провести интерактивное интервью для создания профиля туриста на основе структурированного шаблона.

## Алгоритм работы

### 1. Подготовка
- Прочитай файл `tourist_profile_template.md` из папки `C:\Users\vrech\.claude\cowork\My travels\Profiles\`
- Убедись, что понимаешь структуру всех разделов

### 2. Сбор информации

Задавай вопросы **блоками по разделам** в этом порядке:

#### Блок 1: Базовая информация (ОБЯЗАТЕЛЬНО)
Спроси поочередно:
- **Имя** *(обязательно)*
- **Пол** *(обязательно, варианты: М/Ж)*
- **Дата рождения или возраст** *(обязательно, ДД.ММ.ГГГГ или просто число)*
- Место проживания
- Родной язык
- Email
- Телефон

**Обработка:** Проверь, что имя, пол и возраст/дата заполнены. Если какое-то из этих трёх отсутствует — попроси заполнить снова.

#### Блок 2: Интересы и хобби
- Спроси про любимые виды деятельности (можно выбрать из списка или предложить свои)
- Спроси про текущие хобби (минимум 1-3 дефолтно опциональны)

#### Блок 3: История путешествий
- Посещённые страны/города (таблица: страна, город, дата, впечатления)
- Самое запоминающееся путешествие (место, когда, почему было особенным)

#### Блок 4: Предпочтения в путешествиях
- Стиль путешествий (тип отдыха, частота, продолжительность, бюджет)
- Предпочтения проживания (выбор из списка)
- Предпочтительные сезоны
- Питание (предпочтения и ограничения)
- Как они готовятся к путешествиям (тур, самостоятельно, приложения и т.д.)

#### Блок 5: Текущие и будущие планы
- Планируемые поездки (таблица)
- Список желаний / Bucket List (место и опыты)

#### Блок 6: Языки и навыки
- Языки (язык, уровень A1-C2)
- Специальные навыки для путешествий

#### Блок 7: Документы
- Статусы туристических документов (паспорт, виза, прививки и т.д.)

#### Блок 8: Дополнительные заметки
- Любые другие заметки о туристе

### 3. Подсказки по взаимодействию

**Для опциональных полей:**
- Предложи пропустить вопрос, если ответ "позже": "Можешь пропустить этот вопрос и вернуться к нему позже. Хочешь продолжить?"
- Если пользователь пропускает — отметь это как [Позже] в шаблоне

**Для множественного выбора:**
- Предложи варианты из шаблона
- Позволь добавить свои варианты

**Уточняющие вопросы:**
- Если ответ неясный (например, "много путешествовал") — попроси конкретику
- Для дат спроси в формате ДД.ММ.ГГГГ

### 4. Создание файла

После интервью:

1. **Заполни все поля** в соответствии с полученными ответами
2. **Рассчитай возраст** если была дана только дата рождения
3. **Отметь [Позже]** поля, которые пропустили
4. **Сохрани файл** в папку `C:\Users\vrech\.claude\cowork\My travels\Profiles\` с именем:
   ```
   [Имя туриста]_profile.md
   ```
   Пример: `Иван_Петров_profile.md` или `Maria_Garcia_profile.md`

5. **Текущая дата** в поле "Дата создания профиля": 26 апреля 2026 г.
6. **Дата последнего изменения** = дата создания (или текущей сессии если обновление)

### 5. Обновление существующего профиля

Если профиль туриста уже существует:
- Спроси: "Я нашел существующий профиль [Имя]. Хочешь его обновить или создать новый?"
- Если обновление: перезапроси только те разделы, которые нужно изменить
- Обнови дату последнего изменения

## Пример структуры вопросов для блока

```
📋 **Блок 1: Базовая информация**

1️⃣  Как вас зовут? *
2️⃣  Ваш пол? (М/Ж) *
3️⃣  Ваша дата рождения? (ДД.ММ.ГГГГ) или возраст? *
4️⃣  Где вы проживаете? (можно пропустить)
5️⃣  Ваш родной язык? (можно пропустить)
6️⃣  Ваш email? (можно пропустить)
7️⃣  Ваш телефон? (можно пропустить)
```

* = обязательное поле

## Важные правила

✅ **ВСЕГДА:**
- Проверяй обязательные поля (имя, пол, возраст)
- Предлагай пропустить опциональные вопросы
- Показывай номера и эмодзи для структурированности
- Сохраняй все данные в markdown с таблицами где нужно
- Указывай даты в формате ДД.ММ.ГГГГ

❌ **НИКОГДА:**
- Не требуй все поля сразу
- Не сохраняй без обязательных полей
- Не меняй имя или формат файла без подтверждения
